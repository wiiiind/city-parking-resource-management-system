# 学生毕业实习报告册

## 1. 项目背景介绍

本次毕业实习项目为“城市停车资源管理系统”。项目围绕城市停车资源紧张、停车信息分散、人工管理效率低等问题，设计并实现一个面向管理员、停车场管理员和车主的综合信息系统。系统目标是提升停车资源配置效率，打通停车资源维护、停车业务闭环与运营分析展示。

## 2. 个人承担任务

本人在项目中主要承担后端与数据库主责工作，同时参与系统总体设计与联调测试。具体包括：

- 参与需求分析和系统模块划分
- 设计数据库表结构和业务实体关系
- 完成 Spring Boot 后端接口设计与核心业务逻辑实现
- 联调前端管理端 / 用户端页面
- 编写部署说明、测试用例和技术文档

## 3. 项目成果展示

本项目最终完成了以下成果：

- 搭建前后端分离的城市停车资源管理系统
- 实现停车场、车位、收费规则、车辆档案等基础管理
- 实现车辆入场、出场、费用结算和订单管理
- 实现综合看板和停车资源调度建议
- 完成项目计划书、技术文档、测试用例和部署说明

## 4. 需求分析

系统主要用户包括管理员、停车场管理员和车主。管理员负责系统级资源维护和综合看板查看；停车场管理员负责停车场资源维护和业务处理；车主负责车辆信息维护、停车信息查询和停车记录查看。系统应具备资源管理、业务管理、数据分析和系统支撑四大类功能。

## 5. 系统设计

系统采用前后端分离架构，前端使用 Vue 3，后端使用 Spring Boot，数据库设计基于 MySQL 8。系统共划分为认证、用户、停车资源、停车业务、统计分析、系统公告六大模块。技术文档中进一步给出了用例图、类图、时序图、业务流程图和 E-R 图。

## 6. 系统实现

在实现过程中，我首先完成了数据库设计和后端接口定义，然后实现前端页面和数据展示。为了保证毕业设计答辩时的稳定性，系统提供内置演示数据模式，当前端未连接到本地后端时，仍可继续完成系统演示。后端实现了车辆入场、车辆出场计费、停车记录更新、订单生成和资源统计等核心逻辑。

## 7. 测试与部署

测试工作主要包括登录测试、资源管理测试、车辆进出场测试、费用结算测试、统计页面展示测试和部署验证测试。部署时需要准备 Node.js、JDK 17 和 MySQL 8。前端通过 `npm run dev` 启动，后端通过 `./mvnw spring-boot:run` 启动，数据库通过 SQL 脚本初始化。

## 8. 实习心得

通过本次毕业实习项目，我对软件工程开发流程有了更加完整的理解。从需求分析、系统设计、编码实现到测试部署，每一个阶段都需要提前规划、持续验证和文档沉淀。项目中最大的收获是将课堂知识真正转化为完整系统，并学会从“能运行”进一步走向“能展示、能说明、能交付”。
